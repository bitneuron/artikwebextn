## Quick facts
- **Scoring engine business logic** → see the authoritative file [2026-06-13_scoring-engine-business-logic.md](2026-06-13_scoring-engine-business-logic.md). One-liner: `final = clamp((base − penalties) × archetype_multiplier, 0, 100)`; 6 categories score via **peer-relative percentiles** (S&P 500 sector cohort, daily-cached) with **sector-aware threshold fallback**; archetype (COMPOUNDER/HYPERGROWTH/FINANCIAL/ENERGY/CYCLICAL/TURNAROUND) sets the multiplier from archetype-specific metrics. Skills: `Archetype_Multiplier_Skill.md`, `Peer_Normalization_Skill.md`.
- **(superseded detail below — kept for history)** Scoring is archetype-aware (2026-06-13). `scoring.py` no longer uses one universal ROE/margin rule. `classify_archetype()` → COMPOUNDER / HYPERGROWTH / FINANCIAL / ENERGY / CYCLICAL / TURNAROUND (sector buckets win first: Financial/Energy/Industrials+Materials; then unprofitable→TURNAROUND; then >20% rev in Tech/Comm/Healthcare/ConsumerCyclical→HYPERGROWTH; else COMPOUNDER). `archetype_multiplier()` scores quality with archetype-appropriate metrics; clamped 0.80–1.20. Formula unchanged: `final = clamp((base − penalties) × multiplier, 0, 100)`. `score_ticker_live` returns `archetype` + `multiplier_reason` (shown in artik_broker Explain panel + markdown report).
  - **Real ROIC** = NOPAT/(debt+equity) [opm×rev×0.79 / IC], ROA fallback, clamp ±; replaced the old `0.7×ROE` (which was leverage-inflated — CAT was 36%, now 16%). **Real FCF margin** = freeCashflow/revenue (replaced `fcfm=pm`).
  - **Sector-aware base** — `Inputs` carries `archetype` + `low_margin_model`; `quality_score(i)` branches on them:
    - FINANCIAL: gross-margin→net-margin, owner-earnings→earnings-yield, ROIC→ROE, current-ratio→neutral, D/E>2 penalty suppressed.
    - ENERGY: `quality_score` reweighted to FCF-yield + ROIC/ROCE + debt strength, margins down-weighted (XOM quality 1→8).
    - RETAIL / Consumer Defensive (`low_margin_model`, detected by sector/industry): relaxes gross/net-margin, credits ROIC + FCF conversion (COST 49→57).
    - CYCLICAL: D/E>2 penalty suppressed.
    Results: JPM 72→79 BUY, CAT 49 AVOID→57 HOLD.
  - **IC formula** = Total Debt + Equity − Cash (per spec), with a guard: falls back to Debt+Equity if subtracting cash leaves IC < 10% of revenue (avoids ROIC explosion on cash-rich firms).
  - **Explanation output (#5):** `score_ticker_live` returns `base_metrics_used` / `base_metrics_skipped` per sector → surfaced in artik_broker Explain panel + markdown report.
  - **Trend metrics from multi-year statements (`compute_trend_metrics`):** margin stability (net-margin pstdev <5pp), FCF CAGR, improving profitability, FCF durability (% of yrs FCF>0), FCF conversion (FCF/NI), debt reduction, efficiency ratio (cost/income). Fed into the per-archetype multiplier + explanation (COMPOUNDER stable-margin gate, HYPERGROWTH improving-profit, ENERGY durability, CYCLICAL conversion+stability, TURNAROUND deleveraging/progress). Degrades gracefully (None when statements thin). **Cost: +~0.8s/ticker** (3 extra yfinance statement fetches) — slows the 98-name "Analyze Portfolio" upload; the saved-snapshot Portfolio tab reads CSV so it's unaffected.
  - **Genuinely NOT available (flagged, not faked):** FINANCIAL **capital adequacy** & **credit quality** (need regulatory data, not in yfinance); **efficiency ratio** computes for most companies but returns None for banks whose yfinance income statement lacks an Operating Income line → falls back to "assume efficient". Guardrail-4 "normalize metrics" still not implemented (raw thresholds, no peer percentile).
